# Challenge: Object Analysis and Abstraction in JavaScript

## Objective

The goal of this challenge is to develop your ability to 
**analyze real-world information** and **translate it into a structured JavaScript object** 
using key-value notation.

This is a fundamental skill in software development, as developers constantly 
model real-world entities (users, products, systems) into code.

---

## Scenario

You are given a real-world entity (a person, a product, a vehicle, a book, or 
any similar subject).

Your task is to carefully **analyze that entity** and identify:

* What information is relevant
* How that information can be structured
* How to represent it clearly using a JavaScript object

---

## Requirements

1. Create a JavaScript object using **clean and consistent key-value notation**.
2. Include at least:

   * 12–20 meaningful properties
   * At least one **nested object**
   * At least one **array property**
3. Use **appropriate data types**:

   * Strings
   * Numbers
   * Booleans
   * Arrays
   * Objects

4. Follow **naming conventions**:

   * Use camelCase for keys
   * Use descriptive and meaningful names

---

## Example Entities (Choose One or Propose Your Own)

* A professional athlete
* A car or motorcycle
* A smartphone
* A book
* A video game character
* A company

---

## Expected Output

Your solution must include:

* A properly declared JavaScript object
* Clean and readable formatting
* Logical grouping of related data

---

## Evaluation Criteria

Your solution will be evaluated based on:

* **Clarity**: Are the properties easy to understand?
* **Structure**: Is the object well organized?
* **Abstraction**: Did you correctly identify relevant attributes?
* **Consistency**: Are naming conventions applied correctly?
* **Completeness**: Does the object represent the entity effectively?

---

## Bonus (Optional)

* Add a method (function) inside the object
* Include deeper nesting (object inside object)
* Normalize data where appropriate

---

## Key Learning Outcome

By completing this challenge, you will improve your ability to:

* Think like a software engineer
* Model real-world problems in code
* Prepare for Object-Oriented Programming (OOP)

---

## Submission Guidelines

* Submit your code as a `.js` file
* Ensure your code runs without errors
* Be prepared to explain your design decisions

---

## Tip

Before coding, take a few minutes to **plan your structure on paper**.
